# DrawLang 课程设计方案（最终版）

# 1. 项目目标

实现一个简单绘图语言解释器。

系统接收绘图语言源程序，对其进行：

- 词法分析
- 语法分析
- 语义分析
- 中间表示生成（JSON IR）
- 图形解释执行

最终调用 Python matplotlib 绘制图形。

---

# 2. 总体架构

项目采用：

```text
双前端 + 统一 JSON IR + Python 图形后端
```

架构如下：

```text
                 DrawLang Source
                         │
        ┌────────────────┴────────────────┐
        │                                 │
        ▼                                 ▼
 Java 手工前端                    LEX/YACC 自动前端
 (Lexer + RD Parser)             (Flex + Bison)
        │                                 │
        └────────────────┬────────────────┘
                         ▼
                  JSON Intermediate IR
                         ▼
                Python matplotlib Backend
                         ▼
                      PNG 图像
```


# 10. JSON IR 设计

JSON IR 是前端与后端之间的统一接口。

示例：

```json
{
  "commands": [
    {
      "type": "line",
      "x1": 50,
      "y1": 50,
      "x2": 300,
      "y2": 100,
      "color": "red",
      "lineWidth": 2
    },
    {
      "type": "circle",
      "cx": 400,
      "cy": 200,
      "radius": 80,
      "color": "blue",
      "lineWidth": 3,
      "fill": null
    }
  ]
}
```

# 11. 词法分析设计

## 11.1 Java 手工 Lexer

职责：

- 扫描字符流
- 生成 Token
- 记录行号列号
- 报告词法错误

核心类：

```text
Lexer
Token
TokenType
```

---

## 11.2 Flex

使用：

```text
lexer.l
```

生成：

```text
lex.yy.c
```

负责：

- Token 正则匹配
- 关键字识别
- 行号维护

---

# 12. 语法分析设计

## 12.1 Java 递归下降

核心函数：

```text
parseProgram()
parseStatement()

parseLineStmt()
parseRectStmt()
parseCircleStmt()
parseTextStmt()

match()
```

---

## 12.2 Bison

使用：

```text
parser.y
```

自动生成：

```text
y.tab.c
```

负责：

- LALR 语法分析
- 语法规约
- AST 构建

---

# 13. 语义分析设计

## 13.1 前端检查

Java 前端和 YACC 前端均检查：

- 参数数量
- 参数类型
- 颜色是否合法
- lineWidth > 0
- radius > 0
- width > 0
- height > 0
- fontSize > 0

---

## 13.2 后端检查

Python 后端再次检查：

- JSON 格式
- 坐标是否越界
- 图形是否超出画布
- matplotlib 是否支持颜色

# 21. 最终结论

最终确定：

```text
不支持 CANVAS
不支持表达式
不支持 repeat
默认固定画布
双前端 + JSON IR + Python 后端
```

该方案：

- 满足课程设计要求
- 工程结构清晰
- 双前端可对比
- AST 简洁
- IR 清晰
- 测试容易
- 风险低
- 易于在 9 天内完成
